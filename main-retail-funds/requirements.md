# main-retail-funds 需求梳理

## 目标

做一个简单、可持续使用的辅助交易系统。

系统的核心流程是：

1. 我每天手动收集资金数据，并写入一个固定文件。
2. 我向 Codex 下达命令，由 Codex 负责整理和组装数据。
3. HTML 页面读取整理后的 JSON 数据，展示分析结果，并给出辅助交易结论。

这个系统不是自动交易系统，而是一个：

- 持续集成
- 持续辅助
- 面向人工决策的简单分析系统

## 系统约束

整个系统只保留 3 个核心文件。

### 文件 1：原始数据文件

用途：
由我每天手动补充和维护原始数据。

要求：

- 数据格式尽量简单，便于人工录入。
- 只负责记录事实数据，不负责复杂计算。
- 每天追加或更新一条记录。

建议内容：

- 日期
- 品种名称或代码
- 主力资金值
- 散户资金值
- 价格

推荐录入约束：

- 使用 Markdown 表格维护，方便人工追加，也方便 Codex 稳定解析。
- 固定字段顺序，不随意增减列。
- 每一行只表示某一天、某一个品种的一条事实记录。
- 日期统一使用 `YYYY-MM-DD`。
- 品种名称或代码统一写一个主键值，例如 `IF`、`AU`、`螺纹钢`，后续不要混用多个别名。
- 主力资金值、散户资金值、价格统一写数字，不带单位、不带逗号。
- 主力资金值和散户资金值允许正负号，例如 `120.5`、`-88.2`。
- 缺数据时先空着，不写“未知”或其他文字。

示例表头：

| 日期 | 品种 | 主力资金值 | 散户资金值 | 价格 |
| --- | --- | ---: | ---: | ---: |
| 2026-04-08 | IF | 126.5 | -48.2 | 3924.6 |

### 文件 2：结构化 JSON 文件

用途：
由 Codex 根据原始数据文件进行整理、组装、补全计算字段，生成给前端使用的标准 JSON 文件。

要求：

- JSON 结构稳定，便于 HTML 直接读取。
- 除原始资金数据外，还应包含计算后的信号字段。
- 每次我发出命令后，由 Codex 负责更新这个文件。

### 文件 3：HTML 分析页面

用途：
读取 JSON 文件，展示分析结果，并输出辅助交易结论。

要求：

- 页面尽量简单直接。
- 重点展示交易上有用的结论，而不是堆积原始数据。
- 能让我每天快速看结论、看信号、看变化强弱。

## 数据处理流程

日常使用流程如下：

1. 我先把当天收集好的数据写入原始数据文件。
2. 我对 Codex 下命令，让 Codex 组装数据。
3. Codex 更新结构化 JSON 文件。
4. 我打开 HTML 页面，查看分析结果和辅助交易建议。

## 核心分析逻辑

系统主要分析两类资金：

- 主力资金
- 散户资金

核心目标是判断：

- 主力是否持续增强或减弱
- 散户是否持续增强或减弱
- 变化方向是否一致
- 变化力度是否在加强或减弱

## 基础信号规则

### 信号 A：做多

满足以下条件时，给出做多候选信号：

- 主力资金连续 3 天增强
- 散户资金连续 3 天减弱

### 信号 B：做空

满足以下条件时，给出做空候选信号：

- 主力资金连续 3 天走弱
- 散户资金连续 3 天增强

## 对“连续 3 天”的理解

连续 3 天增强或减弱，主要用于判断方向。

但交易中，力度通常比方向更重要，所以系统不能只看连续上涨或连续下跌，还要看斜率和加速度。

## 斜率与力度分析

### 1. 一阶斜率

用于看资金变化速度。

- 主力斜率 = 今天主力资金 - 3 天前主力资金
- 散户斜率 = 今天散户资金 - 3 天前散户资金

解释：

- 主力斜率越大，说明主力流入增强越明显。
- 主力斜率越小或越负，说明主力走弱或流出更明显。
- 散户斜率同理。

### 2. 二阶斜率

用于看资金变化是否在加速。

- 主力二阶斜率 = 今天主力斜率 - 昨天主力斜率
- 散户二阶斜率 = 今天散户斜率 - 昨天散户斜率

解释：

- 二阶斜率为正，表示当前变化在加速。
- 二阶斜率为负，表示当前变化在减速或继续恶化。
- 如果主力本来就在流出，且二阶斜率继续变差，则空头信号更强。

## 组合信号规则

系统不应只看某一边，而应组合主力和散户的变化共同判断。

### 做多候选

满足以下特征时，应增强做多提示：

- 主力正斜率变大
- 散户负斜率变大

这说明：

- 主力流入在增强
- 散户流出在增强

### 做空候选

满足以下特征时，应增强做空提示：

- 主力负斜率变大
- 散户正斜率变大

这说明：

- 主力流出在增强
- 散户流入在增强

### 背离预警

满足以下特征时，应给出预警：

- 价格上涨
- 但主力斜率走弱

这说明价格表现和主力行为不一致，可能存在背离风险。

后续还可以补充更多背离场景，例如：

- 价格下跌，但主力斜率转强
- 价格创新高，但主力连续减弱

## HTML 页面需要展示的内容

页面至少应展示以下内容：

- 最新日期的主力资金、散户资金、价格
- 连续 3 天方向判断
- 主力一阶斜率
- 散户一阶斜率
- 主力二阶斜率
- 散户二阶斜率
- 当前属于做多候选、做空候选还是背离预警
- 一句简洁的辅助交易结论

## 结论输出形式

HTML 页面最终应尽量输出可以直接阅读的结论，例如：

- 当前偏多：主力连续增强，散户连续减弱，且主力流入斜率继续扩大
- 当前偏空：主力连续走弱，散户连续增强，且主力流出正在加速
- 当前观望：方向不一致，或力度不足
- 当前预警：价格与主力资金变化出现背离

## Codex 需要承担的工作

当我发出命令后，Codex 负责：

- 读取原始数据文件
- 校验数据格式
- 生成或更新结构化 JSON 文件
- 计算连续 3 天方向
- 计算一阶斜率
- 计算二阶斜率
- 根据规则生成信号字段
- 让 HTML 页面可以直接读取结果

## 暂定范围

当前先只做最小可用版本，不追求复杂功能。

第一阶段先实现：

- 单一原始数据文件
- 单一 JSON 输出文件
- 单一 HTML 页面
- 基础信号判断
- 斜率与二阶斜率计算
- 简单的辅助交易结论

## 后续可扩展方向

后面如果有效，再考虑扩展：

- 支持多个品种
- 支持更多信号规则
- 支持图表展示
- 支持历史信号回看
- 支持更细的风险提示

## 当前结论

这是一个三文件的小系统：

- 一个文件由我负责每天录入原始数据
- 一个文件由 Codex 负责组装成结构化 JSON
- 一个 HTML 文件负责读取 JSON 并输出辅助交易结论

核心分析不只看连续 3 天方向，还要重点看：

- 一阶斜率
- 二阶斜率
- 主力与散户的组合关系
- 价格与主力行为是否背离
